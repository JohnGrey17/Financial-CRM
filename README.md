# Budget Planner

## Загальна ідея

Budget Planner — це веб-додаток для планування, погодження та контролю бюджетів компанії.

Основна ціль системи — забезпечити прозорий контроль витрат від рівня окремого проекту до рівня департаменту.

---

# Архітектура системи

```text
Департамент
    ↓
Відділ
    ↓
Проект
    ↓
Платіж
```

## Департамент

Містить набір відділів.

Приклад:

- IT департамент
- Маркетинг департамент
- Операційний департамент

---

## Відділ

Містить набір проектів.

Приклад:

- Інфраструктура
- Розробка
- Маркетинг

---

## Проект

Містить необмежену кількість платежів.

Приклад:

- CRM
- ERP
- Mobile App
- AWS Infrastructure

---

## Платіж

Мінімальна одиниця бюджету.

Поля:

- Назва платежу
- Дата платежу
- Сума
- Коментар
- Статус погодження

Приклад:

```text
AWS сервер
15.05.2026
150 000 грн
```

---

# Агрегація бюджетів

## Проект

```text
Сума проекту =
SUM(усіх платежів проекту)
```

---

## Відділ

```text
Сума відділу =
SUM(усіх проектів відділу)
```

---

## Департамент

```text
Сума департаменту =
SUM(усіх відділів департаменту)
```

---

# Статуси платежу

```text
Чернетка
↓
На погодженні керівника відділу
↓
На погодженні керівника департаменту
↓
Погоджено
```

або

```text
Відхилено
```

---

# Ролі системи

## Співробітник

Може:

- створювати платежі
- редагувати власні платежі
- відправляти на погодження

---

## Керівник відділу

Може:

- бачити всі платежі свого відділу
- погоджувати платежі
- відхиляти платежі

---

## Керівник департаменту

Може:

- бачити всі платежі департаменту
- погоджувати платежі після погодження керівником відділу
- переглядати бюджети відділів
- переглядати бюджети проектів

---

# Основні показники

Для кожного рівня системи:

- Планова сума
- Погоджена сума
- На погодженні
- Відхилена сума

# План реалізації

## Ітерація 1 — Базова структура

### Ціль

Побудувати ієрархію.

Реалізувати:

- Department
- Division
- Project

Можливості:

- створення
- редагування
- перегляд

---

## Ітерація 2 — Платежі

### Ціль

Додати платежі до проектів.

Реалізувати:

Payment:

- Назва
- Дата
- Сума

Можливості:

- створення платежу
- перегляд платежів проекту

---

## Ітерація 3 — Підрахунок бюджетів

### Ціль

Автоматичне обчислення сум.

Реалізувати:

- сума проекту
- сума відділу
- сума департаменту

Dashboard:

- Планова сума
- Загальна сума

---

## Ітерація 4 — Погодження

### Ціль

Запровадити workflow погодження.

Статуси:

```text
DRAFT
PENDING_DIVISION_APPROVAL
PENDING_DEPARTMENT_APPROVAL
APPROVED
REJECTED
```

Можливості:

- погодити
- відхилити

---

## Ітерація 5 — Dashboard

### Ціль

Побудувати сучасний інтерфейс.

Сторінки:

- Dashboard
- Департаменти
- Відділи
- Проекти
- Платежі
- Погодження

---

## Ітерація 6 — Звіти

### Ціль

Аналітика.

Звіти:

- По департаменту
- По відділу
- По проекту

Фільтри:

- Період
- Статус
- Відділ
- Проект

---

## Ітерація 7 — План / Факт

### Ціль

Контроль бюджету.

Для проекту:

```text
План: 1 000 000 грн
Факт: 850 000 грн
Залишок: 150 000 грн
```
